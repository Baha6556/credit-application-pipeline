# Датасет для тестового задания: пайплайн обработки кредитных заявок

Файл: `credit_applications_test_dataset.csv`
200 строк, 16 столбцов. Кодировка UTF-8 (с BOM — открывается корректно в Excel).

## Описание полей

| Поле | Тип (ожидаемый) | Описание |
|---|---|---|
| `client_id` | integer | Идентификатор клиента |
| `full_name` | string | ФИО |
| `birth_date` | date (YYYY-MM-DD) | Дата рождения |
| `national_id` | string | Номер удостоверения личности |
| `phone` | string | Телефон в формате +992XXXXXXXXX |
| `email` | string | Email |
| `monthly_income` | number | Ежемесячный доход (сомони) |
| `employment_status` | enum: employed / self_employed / unemployed / retired / student | Статус занятости |
| `employment_duration_months` | integer | Стаж на текущем месте, мес. |
| `requested_amount` | number | Запрашиваемая сумма кредита (сомони) |
| `requested_term_months` | integer | Запрашиваемый срок кредита, мес. |
| `existing_loans_count` | integer | Кол-во текущих кредитов |
| `region` | string | Регион |
| `marital_status` | enum: single / married / divorced / widowed | Семейное положение |
| `dependents_count` | integer | Кол-во иждивенцев |
| `application_date` | date (YYYY-MM-DD) | Дата подачи заявки |

## Важно для кандидата

Датасет намеренно содержит невалидные и "грязные" данные — это часть задания.
Ваш пайплайн должен сам обнаруживать и корректно обрабатывать (отклонять/логировать/очищать
в зависимости от вашей архитектуры) проблемы вроде:
- отсутствующих обязательных полей;
- некорректных форматов (телефон, email, даты);
- недопустимых числовых значений (отрицательные, нулевые, нечисловые, аномальные);
- значений вне списка допустимых категорий;
- несогласованного форматирования текста (регистр, пробелы);
- дублирующихся идентификаторов клиентов.

Конкретный список проблемных строк не публикуется намеренно — умение находить и
обрабатывать такие случаи и есть часть проверяемого навыка.
